My essays

My last posts depend on each other, and should be read in sequence. Articles not in bold are optional.
  • Assignment Statement Considered Harmfull. Pervasive mutable state make programs more difficult to deal with. While this is old news, most programers don't know or don't care. It's a pity, because simple remedies exist.
  • Defining Syntactic Sugar. I sometimes hear arguments about what is or is not syntactic sugar. This is an attmpt at defining it.
  • Class-based Programming as Syntactic sugar. Most of the time, "Object Oriented" actaully mean classes. Classes are nothing more than syntactic sugar, at least in statically typed langauges such as Java and C++. Sugar is not useless, but seeing past it helps us understand classes better.
  • Taboo "OO". The term "OO" is so overloaded that we should stop using it. Better substitutes are "classes" and "prototypes."
  • How Class based Programming Sucks. Classes are vastly sub-optimal. Functional programming is far better.
